Programming Concepts

  • Memory Layout: Understand the memory organization for different languages (C/C++, Java, Python, Rust, JavaScript)
    • Pointers and References: Grasp the concepts of word length, memory addresses, and their differences in various programming languages.
    • volatile: Be prepared to explain the volatile keyword and its use in embedded systems.
  • Linked List:
    • Implement and explain common linked list operations like:
      • Fast and slow pointer method
      • Loop detection
      • Intersection point finding
      • Reversal
      • Summation of two linked lists
  • Design Patterns:
    • Singleton: Understand basic implementation and multi-threaded synchronization challenges.
    • Factory Method: Explain its purpose and benefits.
    • Proxy Pattern: Discuss its use cases.
    • Observer Pattern: Explain how it facilitates communication between objects.
  • Multithreading:
    • Understand concurrency concepts, synchronization mechanisms (mutexes and semaphores), and producer-consumer patterns.
    • Be prepared to solve problems like sequence printing using multiple threads.
  • Exception Handling: Be familiar with exception handling mechanisms in your preferred programming languages.
  • Secure Coding Guidelines: Understand and explain secure coding practices for embedded systems.
  • Inter-Process Communication (IPC):
    • Explain different IPC mechanisms like shared memory, sockets (Unix sockets), message queues, and event notification.
  • Intra-Process Communication (RPC): Understand the concept of Remote Procedure Call (RPC) and its variants like gRPC.

Hardware Memory

  • RAM (Random Access Memory): Understand its characteristics and role in embedded systems.
  • ROM (Read-Only Memory): Explain its usage for storing programs.
  • Flash Memory: Understand its properties and use cases in embedded systems.
  • Cache Memory: Explain the purpose and benefits of cache memory.
  • DDR (Double Data Rate): Be familiar with DDR memory types and their advantages.

Hardware Interrupts and ISR (Interrupt Service Routine)

  • Explain how hardware interrupts work and the concept of ISRs.

GPIO (General Purpose Input/Output)

  • Understand GPIO functionality and explain how to toggle a GPIO pin.

Protocols

  • Bus Protocols: Familiarize yourself with common bus protocols (e.g., I2C, SPI, UART, CAN). Be prepared to discuss specific protocols like I2C, SPI, UART, RS232, I3C, SMBus, A2B.
  • Serial Protocols, Parallel Protocols, Multimedia Protocols, Communication Protocols, Automotive Protocols (CAN): Gain a general understanding of these protocols.

RTOS Concepts

  • Understand the concept of Real-Time Operating Systems (RTOS) and their role in embedded systems.
  • Be familiar with basic RTOS concepts like tasks, scheduling, and semaphores.

Architecture

  • Bare-metal: Understand the concept of bare-metal programming and examples like Contiki and FreeRTOS.
  • Micro-services: Gain a general understanding of the micro-services architecture and an example like QNX.

Very Basics of Assembly Language

  • Have a basic understanding of assembly language concepts and instructions.

ARM Optimization

  • Understand the basics of ARM optimization techniques for embedded systems.

JTAG (Joint Test Action Group)

  • Explain the purpose and usage of JTAG for debugging embedded systems.

System-on-Chip (SoC)

  • Understand the concept of SoCs and their integration of various components on a single chip.

Microprocessor

  • Explain the basic functionalities of a microprocessor.

C Program Execution

  • Describe the basic steps involved in executing a C program.

Test Frameworks

  • Be familiar with common embedded system test frameworks like CUnit and GTest.

Code Coverage

  • Understand the concept of code coverage and its importance in testing.
